# Crates permitted to use `#![deny(unsafe_code)]` + a scoped `#[allow(unsafe_code)]`
# instead of `#![forbid(unsafe_code)]` (Spec 01 audit A-7).
#
# One crate name per line. `#` starts a comment. Every crate listed here MUST
# confine its `unsafe` to a narrowly-scoped `#[allow(unsafe_code)]` item with a
# SAFETY justification. A crate NOT listed here must carry `#![forbid(unsafe_code)]`
# at its root — so a new crate cannot introduce unsafe without a reviewed entry.
#
# The three suite binaries need exactly one unsafe item: the Android
# NativeActivity `android_main` FFI entry point, generated by
# `loki_app_shell::android_main!` (which emits the scoped `#[allow(unsafe_code)]`).
# The `#[unsafe(no_mangle)]` attribute alone (Rust 2024) makes `forbid` impossible
# for these cdylibs.
