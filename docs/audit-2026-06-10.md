# Codebase Audit — 2026-06-10

Full-workspace audit covering security, correctness, functionality, dead code,
and conventions. Scope: all 17 workspace crates (~90k lines, 444 `.rs` files)
plus `patches/`. Build health at audit time: `cargo check --workspace` and
`cargo clippy --workspace -- -D warnings` both pass cleanly.

Severity scale: **Critical** (active data destruction / remote-triggerable
crash), **High**, **Medium**, **Low**.

---

## 1. Security (untrusted-document attack surface)

XML stack is quick-xml (0.36/0.39), archive stack is zip 2.4.2.

### S1. HIGH — No decompression limits on ZIP/OPC parts (zip bomb → OOM)
- `loki-opc/src/zip/read.rs:74` (and `:30` for `[Content_Types].xml`) —
  `file.read_to_end(&mut data)` per entry, uncapped.
- `loki-odf/src/package.rs:234, 264, 320` — same for mimetype, every part,
  and **every** `Pictures/` image (compounding).
- A few-KB crafted `.docx`/`.odt` can inflate to multi-GB allocations.
  zip 2.4.2 does not cap this; the caller must.
- Fix: read via `Read::take(MAX)` with per-entry and aggregate budgets;
  never pre-allocate from the declared entry size.

### S2. HIGH — ODS repeat-count amplification (memory/CPU bomb + overflow)
- `loki-odf/src/ods/import.rs:86-98, 130-144, 176-191` —
  `number-rows-repeated` / `number-columns-repeated` (`u32`, attacker-
  controlled) drive nested per-cell `HashMap::insert` loops. A tiny file can
  request ~10¹² inserts. Real ODS files legitimately use `1048576` for
  trailing rows, so a naive bound is not enough — styling-only repeats must
  be kept as ranges, and counts clamped to the used range.
- Secondary: `row_idx + current_row_repeated` etc. (`:130,144,176,191,197`)
  overflow → panic in dev builds (overflow-checks on), wrap in release.
  Use `checked_add`/`saturating_add`.

### S3. HIGH — ODT allocation bombs from single attributes
- `loki-odf/src/odt/mapper/document.rs:568` — `repeat_n(spec,
  columns_repeated)` with unclamped `u32` (`number-columns-repeated=
  "4294967295"` → ~4G `ColSpec` allocation).
- `loki-odf/src/odt/reader/document.rs:328` + `mapper/document.rs:310` —
  `<text:s text:c="4294967295"/>` → `" ".repeat(...)` → 4 GB `String`.
- Fix: clamp both to a sane ceiling (e.g. low thousands).

### S4. MEDIUM — Unbounded recursion → stack overflow (ODT)
- `loki-odf/src/odt/reader/document.rs` — `read_inline_children` (`:149`)
  recurses per nested `<text:span>`/`<text:a>`; `read_list` ↔
  `read_list_item` (`:993`) recurse per nested `<text:list>`. The `depth: u8`
  only caps the recorded level, not the recursion. Thousands of nested
  elements abort the process. (DOCX reader is flat — not affected.)
- Fix: thread a real depth counter, error past ~100.

### S5. MEDIUM (needs runtime verification) — `patches/blitz-net` fetches `file:`/`http(s)` URLs
- `patches/blitz-net/src/lib.rs:51-73` — the resource provider reads
  arbitrary `file://` paths (`std::fs::read`) and fetches arbitrary
  http(s)/data URLs. The OOXML/ODF importers do not call it, but if
  document-derived URLs (external images/relationships) ever reach the
  Blitz DOM resource pipeline, this is local-file read + SSRF. Drop the
  `file:` scheme and gate external fetches.

### S6. LOW — Supply chain: branch-pinned git dependency
- Root `Cargo.toml:10` — `loki-file-access = { git = ..., branch = "main" }`.
  Locally `[patch]`-overridden today, but the upstream source should be
  pinned to a `rev =` commit.

### Verified non-issues
- **No XXE / billion-laughs**: quick-xml never resolves DTD/external
  entities; DOCTYPE events are ignored by all parsers.
- **No zip-slip**: `PartName::new` (`loki-opc/src/part/name.rs`) rejects
  `..`/`.`/encoded separators; parts live in an in-memory map — nothing is
  extracted to disk by entry name.
- **No hardcoded secrets**; no document-reachable `unsafe`; `loki-fonts`
  only embeds bundled fonts (no untrusted font parsing);
  `patches/loki-file-access` zenity invocation uses discrete args (no shell
  injection); link opening is scheme-allow-listed (`http/https/mailto`).

## 2. Correctness — Critical data-loss cluster (Loro CRDT bridge)

The editor pipeline is `document_to_loro` on open → mutate →
`loro_to_document` re-derive, so these fire after a **single edit**.

### C1. CRITICAL — Lists, tables, figures, blockquotes become horizontal rules
- Write side: `loki-doc-model/src/loro_bridge/write.rs:67-84` —
  `BulletList`, `OrderedList`, `Table`, `Figure`, `BlockQuote` (and the `_`
  arm) write **nothing**, not even `KEY_TYPE` (verified in source).
- Read side: `loki-doc-model/src/loro_bridge/read.rs:90-106` — list/figure
  stubs and the empty-`KEY_TYPE` `_` arm return `Block::HorizontalRule`.
- Net: open a document containing any of these block types, type one
  character, and each such block is replaced by a horizontal rule. Broader
  than the documented `tab_stops`/`background_color` tech debt.

### C2. CRITICAL — Inline text of many variants deleted on first CRDT write
- `loki-doc-model/src/loro_bridge/inlines.rs:67-72, 89` — the `_` arm of
  `map_inlines` → `extract_plain_text` returns nothing for `Strikeout`,
  `Superscript`, `Subscript`, `SmallCaps`, `Quoted`, `Cite`, `Code`, `Math`,
  `Link`, `Image`, `Note`, `Field`, `Span`, `RawInline` — link text, code,
  footnote bodies etc. vanish.

### C3. HIGH — `split_block`/`merge_block` strip formatting and style identity
- `loki-doc-model/src/loro_mutation/block.rs:58-109, 140-149` — split/merge
  move the tail as plain text (marks dropped: Enter mid-bold strips bold),
  and `split_block` copies para props but not `"style_id"`,
  `KEY_HEADING_JC`, `KEY_HEADING_STYLE`.

### C4. MEDIUM — Byte offsets passed to Loro's Unicode-position cursor API
- `loki-doc-model/src/loro_bridge/mod.rs:230-263` — callers pass UTF-8 byte
  offsets but `LoroText::get_cursor` takes Unicode positions (loro 1.11.1).
  Wrong/None stable cursors for any non-ASCII text. Fix:
  `text.convert_pos(byte, PosType::Bytes, PosType::Unicode)`.

### C5. MEDIUM — More silently dropped round-trip fields
- `loki-doc-model/src/loro_bridge/inlines.rs:95-167, 192-197` — no mark keys
  for `background_color`, `font_name_complex`, `font_name_east_asian`,
  `font_size_complex`; `reconstruct_inlines` always sets `style_id: None`
  (named character styles dropped). Confirmed the documented
  `tab_stops`/para-`background_color` Debug-string writes (`write.rs:184-188`).
- LOW: `HighlightColor::None` Debug-encodes as `"None"` which the decoder
  rejects (`decode.rs:49-69`) — explicit highlight removal is dropped;
  `CodeBlock` attrs discarded; `Block::Plain` comes back as `Block::Para`.

## 3. Correctness — layout & rendering

### R1. HIGH — Infinite loop in paragraph splitting (hang + unbounded allocation)
- `loki-layout/src/flow_para.rs:544-612` — when no line fits a fresh page
  and `space_before > 0`, the flush arm re-runs forever (cursor_y is reset
  to `space_before`, never 0, so the force-split arm at `:555` is
  unreachable), pushing a new `LayoutPage` per iteration. Trigger: one line
  taller than (page height − space_before). Fix: a flushed-once flag.

### R2. MEDIUM — Crash vectors reachable from corrupt files / remote peers
- Style parent-chain cycles: `loki-doc-model/src/style/catalog.rs:98-123`
  and `loki-layout/src/resolve.rs:252-263` recurse on `style.parent` with no
  cycle/depth guard → stack overflow on `A.parent=B, B.parent=A`.
- `loki-primitives/src/color/document.rs:85-92` — `from_hex` byte-length
  check then byte slicing panics on multibyte UTF-8 (e.g. `"#1é345"`);
  reachable from CRDT-synced strings (`props_read.rs:267-277`).

### R3. MEDIUM — Fidelity bugs
- Multi-tab lines: `loki-layout/src/para.rs:736-792` — tab widths computed
  from stale probe positions; second and later tab stops land wrong.
- Images: `loki-vello/src/image.rs:63-74` — translation-only transform;
  layout rect size and display scale never applied → images paint at
  intrinsic pixel size over neighboring content, no HiDPI scaling.
- keep-with-next chains: `loki-layout/src/flow_para.rs:354-411` — chain
  placement bypasses image/footnote/list-marker logic and uses a throwaway
  note counter → inline images, footnote bodies, and bullets/numbers lost
  inside chains; note numbering desyncs.

### R4. LOW
- List `w:start="0"` never increments (`flow.rs:159-174` — 0 used as
  uninitialized sentinel).
- Mixed page sizes: `loki-vello/src/scene.rs:186-187, 408-461` use the first
  section's `page_size` for all page chrome.
- `space_before` lost after explicit page breaks (`flow_para.rs:141-145`).
- DPI inconsistency between render paths
  (`loki-renderer/src/doc_page_source.rs:34-36, 220-228` vs
  `page_paint_source.rs:198`); A4 fallback constants in px vs pt.
- Pointer-keyed font caches (`loki-layout/src/para.rs:844-849`,
  `appthere-canvas/src/font_cache.rs:34,49`) — ABA address reuse risk, no
  eviction; `get_coords` returns default axes (variable-font weights render
  at default on this path).
- Leftover `eprintln!` debug spam: `loki-layout/src/para.rs:863-877`
  ("CALIBRI LAYOUT RUN").

## 4. Functionality (apps & UI)

### F1. HIGH — Presentation editor is a hardcoded demo
- `loki-presentation/src/routes/editor/editor_inner.rs:36-80` — opening a
  real .pptx/.odp shows 3 hardcoded marketing slides under the real
  filename; no load pipeline, no Save anywhere; edits silently lost.

### F2. HIGH — Recents "Delete file" / "Open a copy" are silent no-ops (all 3 apps)
- `loki-text/src/routes/home.rs:181, 186-216` (and the same code in
  `loki-spreadsheet/src/routes/home.rs:156-185`,
  `loki-presentation/src/routes/home.rs:154-183`) — the stored "path" is a
  base64-encoded `FileAccessToken` (see
  `patches/loki-file-access/src/token.rs:135-145`), so
  `fs::remove_file`/`fs::copy` on it always fail; errors swallowed by
  `let _ =`. "Delete file" removes the recents entry but never the file.
- Fix: add `delete()`/`copy_to()` to `FileAccessToken` (also needed for
  Android content URIs) and surface failures.

### F3. HIGH — loki-text loses unsaved edits on tab switch; dirty flag never set
- `loki-text/src/routes/editor/editor_path_sync.rs:44-63` — route change
  drops the `LoroDoc` (all unsaved edits) and reloads from disk; no per-tab
  retention; `shell.rs:99-135` closes tabs without confirmation.
- `is_dirty` is only set at creation (`new_document.rs:54`, `home.rs:74`) —
  no mutation path sets it, no save clears it, so the tab "•" indicator is
  permanently wrong. (Spreadsheet handles this correctly.)

### F4. HIGH — Untitled documents cannot be saved in loki-text
- `editor_save.rs:51-55` errors with "use File → Save As", but no Save As
  exists in loki-text (spreadsheet has one via `pick_file_to_save`); no
  Ctrl+S binding (`editor_keydown_ctrl.rs` handles a/b/i/u/z/y only).
  A blank document can never be persisted.

### F5. HIGH — Scroll-settle/retier pipeline wired to dead channels
- `loki-renderer/src/renderer_state.rs:47` — receiver dropped at creation;
  `document_view.rs:203-211` — a second unrelated channel whose sender drops
  every render; the detector isn't a real hook so a fresh task spawns every
  render (and `DocumentViewProps::eq` is hardwired `false`).
  `RendererState::on_settle()` (cache retiering) can never fire.

### F6. MEDIUM
- Hooks inside conditionals/loops:
  `appthere-ui/src/components/home_tab/recent_files.rs:45,96,219` —
  `use_signal` per row/branch; hook order shifts when the recents list
  changes length → hover state mis-association. Move into child components.
- Hit-testing: `loki-text/src/routes/editor/editor_state.rs:112-113` —
  `window_width` (1280.0) and `scroll_offset` (0.0) never updated; drag
  selection and touch placement wrong after scroll/resize
  (`editor_pointer.rs:62,116,181`).
- Typing/Backspace ignore the active selection (`editor_keydown.rs:63-174`)
  — typed text inserts into the middle of a still-highlighted selection; no
  clipboard handling at all.
- Dead UI: ribbon tab selection (`editor_inner.rs:523-525`), zoom controls
  (all 3 apps), ribbon collapse toggle (spreadsheet/presentation), Home
  template cards 1–4.
- Spreadsheet: save errors logged but invisible to the user
  (`editor_inner.rs:361-412`); formula engine supports only `SUM` and
  `+`/`-` and shows `0` for anything else; grid hardcoded to A1:J30 (data
  outside is invisible/uneditable, though preserved on save).
- Synchronous save/load on the UI thread in loki-text
  (`editor_ribbon.rs:82-89`, `editor_load.rs:52`).
- `patches/dioxus-native-dom` comment overstates coverage: only touch is
  implemented; composition/scroll/wheel/pointer/clipboard remain
  `unimplemented!()` — yet `loki-renderer/src/document_view.rs:221,259`
  registers `onscroll` handlers (panics if Blitz ever delivers one).
- i18n: `loki-i18n/src/loader.rs:46-51` — locales like `en-US-posix` skip
  the fallback and render raw keys everywhere. (Otherwise i18n is sound:
  `fl!()` before `init()` falls back safely; all 96 used keys exist.)

### F7. LOW
- `AtHomeTab` responsive layout dead (`viewport_width` fixed at 375.0).
- Index-based list keys (tab bar, recents, slide thumbnails) re-associate
  state on removal; presentation doesn't fix `active_slide_idx` when
  deleting a slide before the active one.
- Status-bar word count permanently empty; page number stuck at 1
  (documented upstream blocker).
- Safe-area insets in a `OnceLock` — never update on rotation
  (`safe_area.rs:32-42`).
- Debug leftovers in patches (`patches/dioxus-native/src/dioxus_application.rs:100-113`
  `[LOKI/head]` logs; vendored `println!`/`dbg!` in blitz-net/blitz-dom).
- `patches/blitz-shell/src/window.rs:610` — touch end/cancel uses
  `buttons ^= Main` (XOR); unpaired cancel *sets* the bit. Use `&= !Main`.

## 5. Dead code & conventions

- **~~Unwired crate~~ → integrated**: `loki-render-cache` is now the canonical
  cache implementation. `appthere-canvas` re-exports its `PageCache`,
  `CacheTier`, tier policy, scroll state, key trait, and GPU texture utilities
  rather than carrying duplicate copies, so the renderer's existing
  `appthere_canvas::*` paths resolve to it unchanged. Paired with the F5 fix
  below, off-screen pages now downsample by viewport distance, bounding
  resident texture memory instead of keeping the whole document at full
  resolution.
- **Empty module**: `loki-text/src/components/mod.rs` (leftover from the
  `document_source.rs` removal) still exported from `lib.rs:15`. The
  CLAUDE.md known-exceptions table is stale (references the deleted file).
- **Unused export**: `AtTitleBar`/`AtTitleBarProps` never instantiated.
- **Tracked junk at repo root**: `scratch.rs`, `diff_flow.txt`, `iris.png`,
  `iris_output.png`, `iris_stderr.log`, `output.png`, `test_log.txt`,
  `test_log_utf8.txt`, `test_output.png`, `test_output.txt`,
  `test_output_utf8.txt`.
- **300-line ceiling**: 47 non-exception files over the limit (worst:
  `loki-odf/src/odt/reader/document.rs` 1699,
  `loki-odf/src/odt/reader/styles.rs` 1441, `loki-layout/src/flow.rs` 1254,

  1169, `loki-spreadsheet/.../editor_inner.rs` 1148).
- **SPDX headers**: `loki-opc` is deliberately MIT — it will be released as

  `loki-opc/tests/package_tests.rs` and `scratch.rs` have no header at all.
- **unwrap/expect in non-test library code**: 10 total (loki-opc 6 —
  `part/name.rs:108`, `relationships/location.rs:16,21`, `package.rs:174`,
  `zip/read.rs:101,103`; loki-i18n 2 — `loader.rs:42,47`; loki-fonts 1 —

  practice but convention violations.
- **`anyhow`**: none anywhere — clean.
- **Hardcoded UI strings** (i18n violations): loki-text style editor
  (`editor_style_editor.rs:311,382-410` — "Name", "Size (pt)",
  Left/Center/Right/Justify), heading display names
  (`editor_style.rs:48-95`), save errors (`editor_save.rs:63,69`),
  generated style names (`editor_style_catalog.rs:27-42`); spreadsheet and

- **Crate-level clippy allows** in `loki-odf/src/lib.rs:40-53` (13) and
  `loki-ooxml/src/lib.rs:33-45` (12) contradict the "never crate level"

- **Unused dependencies** (spot-check; confirm with cargo-machete):
  ~35 across 10 crates — worst are `loki-presentation` (13+, copy-pasted
  manifest) and `loki-spreadsheet` (11); also `loki-text` (7 render-stack

- **TODO inventory**: 137 (63 `COMPAT`, 65 tagged `TODO`, 9 untagged — the
  untagged ones violate the `TODO(<topic>)` convention). Landmines:
  hardcoded relationship id `"rId3"` in
  `loki-ooxml/src/docx/write/assembly.rs:162` (collision silently corrupts

- **`#![forbid(unsafe_code)]`**: present in all library crates; absent only

## 6. Recommended priority order

1. **Loro bridge data loss (C1/C2)** — active destruction of user content
   in the core editing path; fix before any other editor work.
2. **Data-loss UX cluster (F1–F4)** — tab-switch edit loss + no Save As +
   no-op delete/copy + demo presentation editor.
3. **Decompression/amplification limits (S1–S3)** and the layout infinite
   loop (R1) — malicious-document DoS.
4. **Crash vectors (R2, S4)** — style cycles, `from_hex` panic, ODT
   recursion depth.
5. **Editing correctness (C3/C4, F6 selection/hit-testing)**.
6. Fidelity (R3, C5), dead-channel retier pipeline (F5), i18n fallback.
7. Hygiene: delete `loki-render-cache` + junk files, fix loki-opc SPDX
   headers, prune unused deps, split oversized files, i18n the remaining
   strings.
